# Git Bisect Session — Day 3

## Objective
Find which commit introduced the bug in divide() function.

## Commands Run
$ git bisect start
$ git bisect bad                    # marked HEAD (commit 8) as bad
$ git bisect good d6594e1           # marked commit 3 as last known good

## Root Cause
divide() was implemented as `return a * b` instead of `return a / b`
Symptom: divide(10, 2) returned 20 instead of 5

## Fix
Used git revert to create a new commit undoing commit 4,
then re-added divide() with correct implementation.
